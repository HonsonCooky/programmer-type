Welcome

In the endless attempt to find a frictionless, keyboard centric, developer setup, I've created myself lot of extra work. This web app is one such solution to the issue of testing a keyboard layout's effectiveness in development based activities.

Keyboard Inputs

Designed with context-driven keyboard navigation in mind (kind of the whole point of the application). Turn off your Vimium and follow the bindings wrapped with braces [X]. Context-driven meaning that if some sub-menu is open, only keys from this menu will be valid next inputs; Anything else will reset the context. You can view the current key combination in the bottom right; Where *base* indicates that you have no context.
The buttons work too, but don't be expecting a flawless experience if you're jumping from mouse interactions to keyboard interactions.
- Press Esc to reset the context.
- Press :q to exit the text editor.
